# Innogram Deployment Guide (Nginx + HTTPS 443 only)

This guide describes production deployment with Docker Compose, Nginx in a container, and a self-signed TLS certificate generated by OpenSSL.

## 1. Prerequisites

- [Docker Engine](https://docs.docker.com/engine/install/)
- [Docker Compose](https://docs.docker.com/compose/install/)
- OpenSSL
- Git

## 2. Prepare environment

1. Clone repository:

```bash
git clone https://github.com/your-username/innogram.git
cd innogram
```

2. Create `.env` in repository root from `.env.example` and set at least:

- `NEXT_PUBLIC_API_URL=https://localhost`
- `NEXT_PUBLIC_WS_URL=https://localhost`
- `FRONTEND_PUBLIC_URL=https://localhost`
- `AUTH_MICROSERVICE_URL=http://auth-api:3001/internal/auth`
- `GOOGLE_CALLBACK_URL=https://localhost/auth/google/callback`

Important notes:

- `NEXT_PUBLIC_*` values are baked into the Next.js build. If you change them, you must rebuild `client-app`.
- `docker-compose.prod.yml` reads variables from the repo root `.env`.

3. Add hostnames locally (example for local/LAN test):

- `localhost`
- `localhost`

Map them to your server IP in hosts file on client machines.

## 3. Generate self-signed certificate (OpenSSL)

The repository includes OpenSSL config with SAN:

- `nginx/openssl-san.cnf`

Generate key and cert in `nginx/certs/`:

```bash
mkdir -p nginx/certs
openssl req -x509 -nodes -newkey rsa:2048 -sha256 -days 825 \
  -keyout nginx/certs/privkey.pem \
  -out nginx/certs/fullchain.pem \
  -config nginx/openssl-san.cnf \
  -extensions v3_req
```

Important:

- Do not commit certificate files to git.
- Browsers will warn until certificate (or CA) is trusted on client machines.

## 4. Start stack

Run production compose:

```bash
docker compose -f docker-compose.prod.yml up -d --build
```

In this setup:

- Nginx is the only public entrypoint.
- Only `443` is published to host.
- `core-api`, `auth-api`, and `client-app` are internal in Docker network.

If you run multiple compose stacks in the same machine, make sure you start the infra from `docker-compose.prod.yml` (it creates containers named `innogram-postgres`, `innogram-redis`, `innogram-rabbitmq` on `innogram_network`).

## 5. Apply database migrations

**From the host (Node + Prisma on your machine, database reachable via `DATABASE_URL` in each app):**

```bash
npm run db:migrate:deploy
```

**Inside already-running Compose containers** (typical production host where Postgres is only on the Docker network; requires Docker CLI):

```bash
npm run db:migrate:docker
```

Or run directly:

```bash
docker exec -it innogram-core npm run db:migrate:deploy
docker exec -it innogram-auth npm run db:migrate:deploy
docker exec -it innogram-consumer npm run db:migrate:deploy
```

`npm run db:migrate` / `db:generate` use Turbo on the host and do **not** call `docker`. If you see `sh: docker: not found`, use the Turbo commands above or install the Docker CLI and use `db:migrate:docker` / `db:generate:docker`.

If `auth_microservice` / `notifications_consumer_microservice` are deployed to an existing DB for the first time, they may require a one-time baseline (Prisma `migrate resolve`) to initialize migration persistence.

## 6. Verify

- API: `https://localhost/api`
- App: `https://localhost`
- Health: `https://localhost/health`

If TLS error appears, import certificate into trusted store on each client machine.

## 7. Nginx upstream stability (avoid random 502)

Docker can change container IPs on recreate. Nginx must resolve service names dynamically via Docker DNS.

This repo’s nginx config is set up for that (`resolver 127.0.0.11` + `proxy_pass` via variables). If you see `502 Bad Gateway` with `connect() failed (111: Connection refused)` to an old upstream IP, check that you are using the current `nginx/conf.d/default.conf` and recreate nginx:

```bash
docker compose -f docker-compose.prod.yml up -d --no-deps --force-recreate nginx
```

## 8. Certificate rotation (OpenSSL)

Before expiration:

1. Re-run OpenSSL command from section 3.
2. Replace `nginx/certs/fullchain.pem` and `nginx/certs/privkey.pem`.
3. Restart Nginx container:

```bash
docker compose -f docker-compose.prod.yml restart nginx
```

## 9. Faster builds (recommended)

This repo uses a monorepo `COPY . .` during image build. Keep your Docker build context small:

- `out/`, `Rika/`, `.next/`, `node_modules/` and TLS certs should be ignored in `.dockerignore`.
- After optimization, build context should be tens of MB, not multiple GB.
